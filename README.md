# atividade-scrum-guilhermekendykikuti

NOME: Sistema organizador de tarefas

DESCRIÇÃO: O sistema sera um organizador de tarefas onde a pessoa poderá criar, editar e deletar tarefas variadas.
Esse sistema também contara com um calendario automatizado salvando as suas tarefas em um calendario que também enviará 
notificações caso algum evento ou data importante esteja chegando. E tudo isso será salvo na conta de usuario da pessoa 
podendo ser feito com a própria conta google da pessoa



Product Owner: O Product Owner ira definir as funções do site atravez de comunicação com o usuario ou clientes(Não precisa ser um desenvolvedor)
Scrum Master: O Scrum Master é o que vai guiar a equipe para executar o projeto dentro das metodologias Scrum funcionando como um gerente ou um supervisor
Time de Desenvolvimento: São os desenvolvedores que vão atuar na produção direta do sistema sendo responsáveis pela entrega das Sprints

Todas as funções serão feitas por: Guilherme Kendy Kikuti



Sprint: As sprints é uma meta de conclusão de uma ou mais funcionalidades em um periodo de tempo

Sprint Planning: é uma reunião para definir as metas, prioridades e tempo para conclusão

Daily Scrum: Reunião feita diariamente para checar o status da produção e saber se é necessario alguma mudança no planejamento

Sprint Review: é uma revisão que acontece a cada fim de sprint, onde os toda equipe junta do cliente fazem a checagem e alinhamento das expectativas em relação ao que está sendo feito podendo gerar mudanças para adaptar o projeto ao que é desejado pelo cliente

Sprint Retrospective: acontece nas reuniões sendo basicamente uma reflexão sobre o que aconteceu nas sprints anteriores podendo gerar melhores abordagens futuras



 Product Backlog: são as funcionalidades definidas para o sistema feito pelo Product owner
 
 Sprint Backlog: é a lista de funcionalidades a serem feitas na sprint sendo definidas por sua prioridade
 
 Incremento: a junção de todas as sprints finalisadas sendo um resultado funcional


Product Backlogs:

Adicionar prioridade às tarefas (alta, média, baixa)
Definir data de vencimento
Criar categorias (ex: escola, trabalho, pessoal)
Filtrar tarefas por categoria
Buscar tarefas por nome
Ordenar tarefas por data, prioridade ou categoria


Prioridades:

Definir data de vencimento
Adicionar prioridade às tarefas
Buscar tarefas por nomes
Criar categorias
Filtrar tarefas por categoria

Escolhi essa ordem primeiro focando em funcionalidades essenciais para organizar melhor as tarefas, como data e descrição.
Depois adicionei melhorias com as prioridade para ajudar na organização.
Por fim, incluí categorias e filtros, que são melhorias para facilitar a visualização e uso do sistema.


Sprints:
1. (1 semana) Medio
Definir data de vencimento
Adicionar prioridade às tarefas
Buscar tarefas por nomes


3. (4 à 5 dias) Facil
Criar categorias
Filtrar tarefas por categoria
